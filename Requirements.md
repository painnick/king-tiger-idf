# RC 탱크 프로젝트 요구사항 명세

본 문서는 `README.md`에 기술된 Bluepad32 기반 게임패드 조작 RC 탱크의 요구사항을 정리한 것이다. 구현·검증 시 단일 진실 공급원으로 `README.md`를 유지하고, 본 문서는 그 추출·구조화 역할을 한다.

---

## 1. 문서 범위 및 목표

| 항목 | 내용 |
|------|------|
| **시스템** | ESP32 상에서 동작하는 RC 탱크 펌웨어 |
| **입력** | 블루투스 게임패드 (Bluepad32) |
| **목표** | 트랙·터렛·포 마운트·포신·기관총·조명·사운드·볼륨·설정 영속화를 게임패드로 제어 |

---

## 2. 용어

| 용어 | 설명 |
|------|------|
| 트랙 | 좌·우측 주행 트랙 모터 (DRV8833) |
| 터렛 | 상부 회전 구동 (DRV8833) |
| 포 마운트 | 포 상하 DC 모터 (DRV8833 #2 B채널, GPIO 19·4) |
| 포신 | 발사 동작용 서보 당김 (GPIO 18) |
| 효과음 N | SD카드의 `000N.mp3`에 대응하는 트랙 |

## 3. 기능 요구사항 (FR)

### 3.1 게임패드 연결

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-GP-01 | Bluepad32를 통해 블루투스 게임패드와 연결할 수 있어야 한다. | 필수 |
| FR-GP-02 | 부팅 후 블루투스 페어링 모드로 진입하고, 게임패드에서 ESP32에 연결할 수 있어야 한다. | 필수 |
| FR-GP-03 | 연결 성공 시 효과음 4(`0004.mp3`)를 재생하고, 대기 중이던 효과음 1 반복 재생을 중단해야 한다. | 필수 |
| FR-GP-04 | 연결 후 게임패드 조작이 가능한 상태가 되어야 한다. | 필수 |

### 3.2 주행·포탑 구동

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-MV-01 | 좌측 스틱 Y축으로 좌측 트랙 전·후진을 독립 제어해야 한다. | 필수 |
| FR-MV-02 | 우측 스틱 Y축으로 우측 트랙 전·후진을 독립 제어해야 한다. | 필수 |
| FR-MV-03 | D-PAD 좌·우로 터렛 좌·우 회전을 제어해야 한다. | 필수 |
| FR-MV-04 | D-PAD 상·하로 포 마운트 DC 모터를 DRV8833 B채널로 구동해야 한다. 스트로크·안전(리미트 스위치·시간·전류·선택적 엔코더 등)은 펌웨어·기계 설계에서 정한다. | 필수 |

### 3.3 발사·조명

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-FX-01 | B 버튼: 포신 당김 동작, **포신 LED 점등**, 효과음 2 재생, 동작 지속 **200ms** (README “기능”·“게임패드 조작법” 기준). | 필수 |
| FR-FX-02 | A 버튼: 기관총 발사, **효과음 3** 재생, **1초간** 유지. LED는 “기관총 발사 시 포신 LED 동작 없음” 및 기능 목록의 “기관총 LED 동작 없음”에 따른다. *(핀 할당 표에는 GPIO 23 기관총 LED가 있으나 동작 설명과 상충 가능 — 구현 시 README 최신본과 하드웨어 의도를 확정할 것.)* | 필수 |
| FR-FX-03 | Y 버튼: 헤드라이트 ON/OFF 토글. 입력 처리 간격 **100ms** (README “헤드라이트” 항목). | 필수 |

### 3.4 오디오·볼륨

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-AU-01 | JQ6500 모듈을 통해 SD카드에 저장된 MP3 효과음을 재생해야 한다. | 필수 |
| FR-AU-02 | L1: 볼륨 감소, R1: 볼륨 증가. 볼륨 값 범위 **10~30**, 버튼 반복 처리 간격 **100ms** (README “기능”·“볼륨 조절” 본문). | 필수 |
| FR-AU-03 | 볼륨 설정은 버튼을 뗀 시점에 EEPROM에 자동 저장되어 다음 부팅 후에도 유지되어야 한다. | 필수 |
| FR-AU-04 | *(참고)* README “게임패드 조작 요약” 표에는 **L1+R1이 아닌 “L1 + (X 또는 Y)” / “R1 + (X 또는 Y)”** 로 기술되어 있어 본문 볼륨 절과 불일치한다. **구현 요구는 본문( L1 / R1 단독 )을 기본으로 하되**, 제품 의도에 따라 표를 수정하거나 구현을 통일해야 한다. | 정합성 |

### 3.5 햅틱(진동) 피드백

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-HP-01 | 포신 발사 시 게임패드 진동 **400ms**. | 필수 |
| FR-HP-02 | 기관총 발사 시 게임패드 진동 **300ms**. | 필수 |
| FR-HP-03 | EEPROM 초기화 수행 시 게임패드 진동 **800ms**. | 필수 |

### 3.6 설정 초기화

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-CF-01 | Select + Start 버튼을 **3초간** 동시 누르면 모든 EEPROM 설정을 초기화하고 ESP32를 재시작해야 한다. | 필수 |

### 3.7 부팅·런타임 동작

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-RT-01 | 대기 상태에서 효과음 1(`0001.mp3`)을 **반복 재생**해야 한다 (게임패드 연결 전·연결 로직과의 관계는 README “게임패드 연결” 절에 따름). | 필수 |
| FR-RT-02 | 볼륨 변경 결과는 시리얼 로그로 확인 가능해야 한다 (README “게임패드 피드백”). | 권장 |

## 5. 하드웨어 인터페이스 요구사항

### 5.1 ESP32 GPIO 할당 (README 표 기준)

| GPIO | 연결 대상 |
|------|-----------|
| 5 | 포신 LED (부팅 스트랩 유의) |
| 12 | (미사용 - 일반 GPIO, 부팅 시 플래시 전압 영향 유의) |
| 13 | (미사용 - 일반 GPIO) |
| 14 | (미사용 - 일반 GPIO, 부팅 시 신호 발생 유의) |
| 15 | (미사용 - 일반 GPIO, 부팅 시 로그 및 신호 발생 유의) |
| 16 | 기관총 LED |
| 17 | 헤드라이트 LED |
| 18 | 포 마운트 IN2 (DRV8833 #2 BIN2) |
| 19 | 포 마운트 IN1 (DRV8833 #2 BIN1) |
| 21 | 터렛 모터 IN2 (DRV8833 #2 AIN2) |
| 22 | 터렛 모터 IN1 (DRV8833 #2 AIN1) |
| 23 | 포신 서보 |
| 25 | 좌측 트랙 IN2 (DRV8833 #1 AIN2) |
| 26 | 우측 트랙 IN1 (DRV8833 #1 BIN1) |
| 27 | 우측 트랙 IN2 (DRV8833 #1 BIN2) |
| 32 | 좌측 트랙 IN1 (DRV8833 #1 AIN1) |
| 33 | JQ6500 RX (ESP32 UART TX) |

### 5.1.1 할당 검토 (요구사항)

| ID | 내용 |
|----|------|
| HW-GPIO-01 | GPIO **6~11**은 내장 플래시 점유로 **사용하지 않는다**. |
| HW-GPIO-02 | GPIO **34~39**는 **출력 불가**이므로 모터·LED 구동에 할당하지 않는다. |
| HW-GPIO-03 | **JTAG 핀(12~15) 사용 유의점**: 본 프로젝트에서는 JTAG를 사용하지 않으나, 일부 핀은 부팅 시 시스템 상태를 결정하는 스트래핑 핀으로 동작하므로 다음 사항에 주의한다.<br>- **GPIO 12 (MTDI)**: 부팅 시 High이면 내부 플래시 전압을 1.8V로 강제하여 부팅 실패 가능성 있음 (가급적 부팅 시 Low 유지 권장).<br>- **GPIO 15 (MTDO)**: 부팅 시 High이면 부팅 로그를 출력하며, 부팅 과정에서 짧은 신호(PWM/펄스)가 출력될 수 있음.<br>- **GPIO 13, 14**: JTAG 미사용 시 비교적 자유로우나, **GPIO 14**의 경우 부팅 시 의도치 않은 신호가 발생할 수 있으므로 민감한 액추에이터 연결 시 주의. |
| HW-GPIO-04 | GPIO **25·26**은 DAC 핀과 멀티플렉스된다. DAC를 쓰지 않는 본 펌웨어 전제에서는 트랙 PWM에 사용 가능하다. |
| HW-GPIO-05 | GPIO **5**(포신 LED)는 리셋 시 **스트랩**으로 샘플링될 수 있다. 부관측 LED로 할당하여 부팅 순간의 전압 레벨이 모터 구동 등 핵심 동작에 영향을 주지 않도록 한다. |
| HW-GPIO-06 | JQ6500 UART는 **ESP32 TX(33) 단방향**으로 제어한다. JQ6500의 TX(상태 값)는 수신하지 않는다. |
| HW-GPIO-07 | 표의 IN1/IN2 명과 DRV8833 절의 AIN1/AIN2·BIN1/BIN2는 **동일 배선**을 의미하며, 구현·검수 시 교차 확인한다. |

### 5.2 전원·접지

| ID | 요구사항 |
|----|----------|
| HW-01 | 좌·우 트랙·터렛·포 마운트 DC가 사용하는 DRV8833의 VM은 **12V**, 논리 VCC는 **3.3V**, GND는 공통 접지. |
| HW-02 | 포신 서보만 **5V** 및 GPIO **23** 신호; 포 마운트는 DRV8833 B채널(GPIO **19·18**)로 구동한다. |
| HW-03 | JQ6500: 모듈 RX→GPIO 33 (단방향 제어), VCC **5V**(보드 매뉴얼 준수), GND 공통, 스피커 출력(SP+/SP− 등)→스피커. |

### 5.3 DRV8833 채널 매핑 (README 그대로)

- **좌측 트랙 (DRV8833 #1 A)**: AIN1/AIN2 → GPIO 32/25
- **우측 트랙 (DRV8833 #1 B)**: BIN1/BIN2 → GPIO 26/27
- **터렛 (DRV8833 #2 A)**: AIN1/AIN2 → GPIO 22/21
- **포 마운트 DC (DRV8833 #2 B)**: BIN1/BIN2 → GPIO 19/18

## 7. 초기화 시퀀스 요구사항

부팅 후 다음 순서를 만족해야 한다 (README “초기화 과정”).

1. EEPROM에서 볼륨 설정 로드  
2. 구동부 초기화: 포 마운트 DC **정지**, 포신 서보 **90°**  
3. JQ6500 초기화 후 효과음 1 재생 시작  
4. Bluepad32 초기화  
5. 게임패드 연결 대기

## 9. README와의 정합성 이슈 (구현 전 클리어 권장)

1. **기관총 LED**: 핀 표(GPIO 16)와 “기관총 시 LED 없음” 서술의 정책 통일.  
2. **조작 요약 표**: A/B 버튼의 LED 설명이 상세 절(기관총 LED 없음)과 다를 수 있음 — 요약 표는 상세 절에 맞게 정정 권장.  
3. **볼륨 키 조합**: 본문은 L1/R1 단독, 요약 표는 L1/R1 + X 또는 Y — 하나로 통일 필요.
